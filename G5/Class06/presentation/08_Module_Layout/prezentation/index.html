HTML5 layout elements

Introduction

  • Webpages can and will look pretty different from one another, but they all tend to share similar standard components
  • In addition to defining individual parts of your page (such as "a paragraph" or "an image"), HTML5 also boasts a number of block level elements used to define areas of your website (such as "the header", "the navigation menu", "the main content column")

Introduction (2)

  • In your HTML code, you can mark up sections of content based on their functionality — you can use elements that represent the sections of content described above unambiguously, and assistive technologies like screenreaders can recognise those elements and help with tasks like "find the main navigation", or "find the main content."
  • To implement such semantic mark up, HTML5 provides dedicated tags that you can use to represent such sections

Headers & Footers

  • <header> represents a group of introductory content. If it is a child of <body> it defines the global header of a webpage, but if it's a child of an <article> or <section> it defines a specific header for that section
  • <footer> represents a group of end content for a page that generally contains fine print, copyright notices, contact info or secondary navigation


<header>
	  <h1>Page main heading</h1>
</header>
					


<footer>
	<p> 2020 © SEDC. All rights Reserved. </p>
</footer>

Navigation

  • The <nav> contains the main navigation functionality for the page. Secondary links, etc., would not go in the navigation
  • Links to the site's main sections; usually represented by menu buttons, links, or tabs
  • Many web designers consider the navigation bar to be part of the header rather than an individual component, but that's not a requirement


<nav>
	<ul>
		<li><a href="">Home</a></li>
		<li><a href="">Our Team</a></li>
		<li><a href="">Projects</a></li>
		<li><a href="">Contact</a></li>
	</ul>

	<form action="" method="">
		<label for="search">Search: </label>
		<input type="search" name="search" placeholder="Search anything"
		id="search" />
		<input type="submit" value="Go!" />
	</form>
</nav>

Main Content

  • The <main> is for content unique to this page. Use <main> only once per page, and put it directly inside <body>.
  • Ideally this shouldn't be nested within other elements


<body>
	<main>

		<!-- Our page's main content -->

	</main>
</body>

Articles

  • The <article> element encloses a block of related content that makes sense on its own without the rest of the page (e.g., a single blog post)

Sections

  • The <section> element is similar to <article>, but it is more for grouping together a single part of the page that constitutes one single piece of functionality (e.g., a mini map, or a set of article headlines and summaries), or a theme
  • It's considered best practice to begin each section with a heading
  • Also note that you can break <article>s up into different <section>s, or <section>s up into different <article>s, depending on the context


<section>
	<article>
		<h2>Japanese Vegetarian</h2>
		<h3>Five week course in London</h3>
		<p>A five week introduction to traditional Japanese
		vegetarian meals, teaching you a selection of rice and noodle dishes.</p>
	</article>
	<article>
		<h2>Sauces Masterclass</h2>
		<h3>One day workshop</h3>
		<p>An intensive one-day course looking at how to create the most
		delicious sauces for use in a range of Japanese cookery.</p>
	</article>
</section>

Aside

  • The <aside> element contains content that is not directly related to the main content but can provide additional information indirectly related to it (glossary entries, author biography, related links, etc.)
  • Usually, it's contextual to what is contained in the main content
  • The <aside> content can also be nested within the main content



<aside>
	<h2>Popular Recipes</h2>
	<a href="">Yakitori (grilled chicken)</a>
	<a href="">Tsukune (minced chicken patties)</a>
	<a href="">Okonomiyaki (savory pancakes)</a>
	<a href="">Mizutaki (chicken stew)</a>
</aside>

Non-semantic wrappers

  • Sometimes you'll come across a situation where you can't find an ideal semantic element to group some items together or wrap some content
  • For cases like these, HTML provides the <div> and <span> elements

Non-semantic wrappers (2)

  • <div> is a block level non-semantic element, which you should only use if you can't think of a better semantic block element to use, or don't want to add any specific meaning
  • As a "pure" container, the <div> element does not inherently represent anything
  • <span> is an inline non-semantic element, which you should only use if you can't think of a better semantic text element to wrap your content, or don't want to add any specific meaning



<div id="shopping-cart">
	<h2>Shopping cart</h2>
	<ul>
		<li>
			<a href="">Chicken: <span id="price">$9.99<span></a>
		</li>
	</ul>
</div>
